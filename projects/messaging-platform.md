# Cloud-Based Messaging Platform

## Goal
Design a backend-heavy messaging platform with reliable message persistence
and low-latency access.

## Key Focus Areas
- API design
- Message ordering
- Horizontal scalability
- Fault tolerance

## Stack
- AWS ECS
- DynamoDB
- Redis
- Kafka

## Engineering Decisions
- Stateless services
- Event-driven message flow
- Cache-first reads for performance
